# 統合議事録生成プロンプト

## 0. ゴール
- **同一日・同一会議の全文を統合**し、
  1) "簡易議事録" (600–700 字) と
  2) "詳細議事録" (800–1000 字)
  の **2 段階で出力** する。
- 文字数・項目数・医療用語表記を厳守し、読みやすいストーリーに再構成する。
---

## 1. 入力
1. **音声文字起こしテキスト**
   - 行頭に `DATE_TIME=YYYY/MM/DD HH:MM` を付与（例 `DATE_TIME=2024/05/10 14:03`）。
   - タグがない場合は前行のタグを継承。
2. **医療用語辞書**（例 `じょくそう→褥瘡`、`ろうけん→老健` など）  病院・介護系のスタッフの会話を念頭において、できる限り想像力を働かせて正しい漢字などに変換してほしい

---

## 2. 前処理 (STEP 0)
1. **同一 `DATE_TIME` タグ** を持つ行を結合し、1 ブロック＝1 会議に統合。
2. 参加者名・職種の表記ゆれを正規化（初出のみフル表記、以降イニシャル可）。
3. 医療用語を正式名称へ一括変換。

---

## 3. 出力 (STEP 1 → STEP 2)

### STEP 1 │ 簡易議事録（600–700 字）
- 構成順：
  1. **会議概要**（会議名／日時／参加者数）
  2. **主要議題**（箇条書き・各行20–25字）
  3. **決定事項**（箇条書き）
  4. **主要アクション**（担当＋期限）
- 句読点・改行でリズムを取り、全体で600–700 字に収める。

### STEP 2 │ 詳細議事録（800–1000 字）
- 必須見出しと制限：
  | 見出し                | 内容・制約                                                    |
  |-----------------------|--------------------------------------------------------------|
  | 会議名                | テキストから抽出／なければ「未指定」                         |
  | 日時                  | YYYY/MM/DD HH:MM ／なければ「未指定」                 |
  | 開催場所              | テキストから抽出／なければ「未指定」                         |
  | 参加者                | 「氏名（職種）」形式で全員列挙。職種不明は「職種不明（推定）」 |
  | 議論内容 *max 12*     | 「発言者名：要点」20–25 字／行                               |
  | 次回までの課題・アクション *max 6* | 「課題内容（担当：氏名、期限：YYYY/MM/DD or なし）」 |
  | 次回会議日程          | YYYY/MM/DD HH:MM ／なければ「未定」                    |
- 全体で800–1000 字（見出し含む）程度
- 文字数超過時は議論要点を**最短20字**まで短縮し調整。

---

## 4. 品質チェック（自動）
1. **文字数**：STEP 1→600–700 字／STEP 2→800–1000 字。
2. **項目数**：議論≤12、課題・アクション≤6。
3. **医療用語・人名**：（例 `じょくそう→褥瘡`、`ろうけん→老健` など）  病院・介護系のスタッフの会話を念頭において、できる限り想像力を働かせて正しい漢字などに変換してほしい
4. 欠損項目は見出しのみ記載（例 `## 次回会議日程\n未定`）。
